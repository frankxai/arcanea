# Contributing to Arcanea

Thank you for your interest in contributing to Arcanea!

---

## Ways to Contribute

### Add to the Mythology

- **New Luminors** - Archetypal guides ([template](./community/luminor-template.md))
- **Bestiary Creatures** - Creative blocks ([template](./community/creature-template.md))
- **Wisdom Texts** - Insights from practice

### Improve the Toolkit

- **Skills** - Frameworks for specific tasks
- **Platform Guides** - Setup for new AI platforms
- **Examples** - Real session transcripts

### Fix and Enhance

- Documentation improvements
- Bug fixes
- Typos and clarity

---

## Getting Started

1. **Fork** the repository
2. **Clone** your fork locally
3. **Create a branch** for your changes
4. **Make your changes** following our guidelines
5. **Test** your changes
6. **Submit** a pull request

---

## Guidelines

### Voice and Tone

Arcanea has a distinctive voice:

- **Elevated but accessible** - Wisdom, not pretension
- **Mythic but practical** - Always actionable
- **Universal** - Avoid cultural appropriation
- **Direct** - No filler or fluff

### Code Style

- Use clear, descriptive names
- Follow existing patterns
- Comment complex logic

### Commit Messages

```
type: brief description

Longer explanation if needed.
```

Types: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`

---

## Pull Request Process

1. Update documentation for any changes
2. Test on at least one platform (Claude Code preferred)
3. Fill out the PR template
4. Request review

---

## Code of Conduct

- Be respectful and constructive
- Welcome newcomers
- Assume good intent
- Create from experience, not theory
