## The Five Pillars in Practice

### Pillar 1: Identity
```
Who should the AI be?
- Role definition
- Expertise level
- Communication style
- Constraints on behavior
```

### Pillar 2: Context
```
What world does this exist in?
- Background information
- Relevant constraints
- Stakeholders
- Success criteria
```

### Pillar 3: Constraint
```
What boundaries shape the work?
- Hard requirements
- Soft preferences
- Format specifications
- Length limits
```

### Pillar 4: Exemplar
```
What does good look like?
- Examples of desired output
- Anti-examples to avoid
- Quality benchmarks
- Style references
```

### Pillar 5: Iteration
```
How will we refine?
- Feedback loops
- Refinement commands
- Quality checkpoints
- Final criteria
```

## Prompt Templates

### Creative Generation
```markdown
# [Project Name]

## Identity
You are [role] with expertise in [domain].
Your voice is [characteristics].

## Context
We are creating [what] for [audience] in [setting].
The goal is [specific outcome].

## Constraints
- Must: [requirements]
- Should: [preferences]
- Avoid: [anti-patterns]
- Format: [structure]

## Exemplar
[One excellent example]

## Request
[Specific ask with success criteria]
```

### Problem Solving
```markdown
# [Problem Name]

## Situation
[What's happening]

## Complication
[What's blocking]

## Question
[What to solve]

## Constraints
[Limitations]

## Success Criteria
[How we know it's solved]
```

### Iteration Refinement
```markdown
# Refinement Request

## Current State
[What we have]

## Gap Analysis
Works: [list]
Missing: [list]
Wrong: [list]

## Direction
More: [quality]
Less: [quality]

## Specific Changes
1. [Change 1]
2. [Change 2]
```
